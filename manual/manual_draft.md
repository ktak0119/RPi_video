# クイックガイド

前提: ユーザー名`pi`・Raspberry Pi OS Bookworm・ルーターで固定IPを設定済み。


### ステップ1: OSの書き込み

1. セットアップ用PCに「Raspberry Pi Imager」をインストールする
   (<https://www.raspberrypi.com/software/>)
2. SDカードをPCに接続し、Imagerを起動する
3. OS: 「Raspberry Pi OS (64-bit)」(Bookworm) を選択する
4. 書き込み先: SDカードを選択する
5. 「OSをカスタマイズする」で以下を設定する
   - ユーザー名: `pi` / パスワード: 任意
   - Wi-FiのSSID・パスワード
   - SSH: 有効化する
6. 書き込みを実行する

### ステップ2: Wi-Fiスリープの無効化（必須）

> これをしないと起動後数分でWi-Fiが切れる。

書き込み完了後、SDカードが自動で取り出された場合は抜き差しして再接続する。

エクスプローラで`bootfs`ドライブを開き、`user-data`を右クリック →
「プログラムから開く」→「メモ帳」で開く。
(macOSの場合: ターミナルで `cd /Volumes/bootfs && nano user-data`)

ファイル末尾に以下を追加して保存する。

```
write_files:
  - path: /etc/NetworkManager/conf.d/wifi-powersave-off.conf
    owner: root:root
    permissions: '0600'
    content: |
      [connection]
      wifi.powersave = 2

runcmd:
  - |
      for i in $(seq 1 60); do
          if /usr/sbin/iw dev wlan0 info >/dev/null 2>&1; then
              /usr/sbin/iw dev wlan0 set power_save off || true
              break
          fi
          sleep 1
      done
```

SDカードをRPiに挿して起動する。

### ステップ3: 固定IPの設定

ルーターの管理画面でRPiに固定IPを割り当てる。
→ 詳細は詳細解説の6章を参照。

### ステップ4: SSHでログイン

セットアップ用PCのターミナル(Windowsの場合はPowerShell)から実行する。

```bash
ssh pi@<RPiの固定IP>
```

「Are you sure you want to continue connecting?」→ `yes` → Enterでパスワードを入力。

### ステップ5: 必要パッケージのインストール

```bash
sudo apt update
sudo apt install -y python3-picamera2 rpicam-apps python3-flask alsa-utils git
```

### ステップ6: Webアプリの取得

```bash
git clone https://github.com/ktak0119/RPi_video.git ~/RPi_video
```

### ステップ7: 自動起動の設定

```bash
sudo cp ~/RPi_video/rpi-webapp.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now rpi-webapp.service
sudo systemctl status rpi-webapp.service
```

最後のコマンドで `active (running)` と表示されれば完了。

### ステップ8: 動作確認

タブレット・スマートフォンのブラウザで以下にアクセスする。

```
http://<RPiの固定IP>:8080/
```

ホーム画面が表示されればセットアップ完了。

# 詳細解説

### 1. システム概要

#### 1.1 全体構成

RPi上でWebアプリ(Flask)が常時動作しており、タブレットやPCのブラウザから
`http://<RPiの固定IP>:8080/`にアクセスすることで、撮影の開始・停止、設定
変更、撮影データの確認・削除、RPiの再起動・シャットダウンなどを、すべて
ブラウザ上の操作だけで行うことができる。

```
[タブレット/PCのブラウザ]  --(Wi-Fi/LAN)-->  [Raspberry Pi]
        http://<固定IP>:8080/                Webアプリ(常時稼働)
                                              + カメラ + (USBマイク)
```

操作する端末にスクリプトなどをインストールする必要はなく、ブラウザだけで
利用できる。

#### 1.2 撮影ループ

[撮影]ページで「撮影を開始する」を押すと、以下を`video_number`回
繰り返す(`video_number`は[設定]ページで指定。8章)。

1. サムネイル撮影 (`rpicam-jpeg`、1枚の静止画)
2. 動画撮影 (`rpicam-vid`、`video_duration`秒)。音声を同時録音する設定の
   場合は、`arecord`でwavファイルも同時に記録する
3. 1.に戻る

途中で「撮影を停止する」を押すと、現在の動画撮影の終了を待たずに、
ただちに撮影を中断して待機状態に戻る(9.4節)。

ファイルの命名規則・PC側での取り扱いは10章を参照。


### 2. 必要なもの

- Raspberry Pi本体(Raspberry Pi OS "Bookworm"が動作するモデル)
- microSDカード(数日以上の撮影を行う場合は256GB以上推奨。1080p/8Mbpsで
  録画すると1時間あたり数GBになるため、録画時間に応じた容量を用意すること)
- Raspberry Pi用カメラモジュール
- USBマイク(音声を同時録音したい場合のみ。任意)
- Wi-Fiルーター(RPiに「固定IP」を割り当てられるもの。固定IPとは、RPiに
  毎回同じIPアドレス〈識別番号〉を割り当てる設定のこと。詳細は6章参照。
  屋外・移動先での使用にはモバイルバッテリーで動作するモバイルルーターが
  便利。詳細は12章参照)
- 操作用端末(ブラウザが使えるタブレット・スマートフォン・PCなど)
- セットアップ用PC(初回セットアップ時のみ)


### 3. RPiの初期セットアップ

すでにRPiが起動していて、Wi-Fi接続・SSH(遠隔ログイン)でのログインができる
場合は3.4節から進めればよい。

#### 3.1 OSの書き込み

1. PCに「Raspberry Pi Imager」をインストールする
   (<https://www.raspberrypi.com/software/>)
2. SDカードをPCに接続し、Raspberry Pi Imagerを起動する
3. OSとして「Raspberry Pi OS (64-bit)」(Bookworm)を選択する
4. 書き込み先としてSDカードを選択する
5. 書き込み前の設定画面(歯車アイコン/「OSをカスタマイズする」)で
   以下を設定しておくと、初回起動後すぐにWi-Fi/SSHで使える
   - ユーザー名・パスワード
     (本マニュアルではユーザー名`pi`を前提に説明する。`pi`以外を
     使う場合は、5章・付録Aの`<user>`を実際のユーザー名に読み替える
     とともに、付録Aの追加設定が必要になる)
   - Wi-FiのSSID・パスワード
   - SSHを有効化する
6. 書き込み完了後、Wi-Fiスリープを無効化する設定をSDカードに書き込む。
   Raspberry Pi Imagerは書き込み完了後にSDカードを自動で取り出す場合がある。
   その場合はSDカードをいったん抜き、再度PCに挿し直してからつづける。

   **この設定をしないと、起動後Wi-Fiに接続しても数分で回線が切れてしまう。**

   エクスプローラで`bootfs`ドライブを開き、`user-data`ファイルを右クリック
   →「プログラムから開く」→「メモ帳」で開く。

   (macOSの場合はターミナルで `cd /Volumes/bootfs && nano user-data`)

   ファイルの**末尾**に以下を追加して保存する。

   ```
   write_files:
     - path: /etc/NetworkManager/conf.d/wifi-powersave-off.conf
       owner: root:root
       permissions: '0600'
       content: |
         [connection]
         wifi.powersave = 2

   runcmd:
     - |
         for i in $(seq 1 60); do
             if /usr/sbin/iw dev wlan0 info >/dev/null 2>&1; then
                 /usr/sbin/iw dev wlan0 set power_save off || true
                 break
             fi
             sleep 1
         done
   ```

   この2つの設定の役割:
   - `write_files`: NetworkManager設定ファイルを作成し、以後の起動でも
     Wi-Fiスリープを常に無効化する
   - `runcmd`: 初回起動時に`iw`コマンドで即座にスリープOFFを実行する
     (NetworkManagerの設定が反映される前の短い空白期間をカバーするため)

   > **注意**: Raspberry Pi Imagerが生成した`user-data`に既に`write_files:`や
   > `runcmd:`セクションがある場合は、末尾に重複して書くのではなく、既存の
   > セクション内にそれぞれの項目を追記すること(YAMLでは同じキーを複数書くと
   > 後のものが無視される)。

7. SDカードをRPiに挿して起動する

#### 3.2 SSHでのログイン確認

SSH(Secure Shell)とは、PCからRPiをネットワーク経由で遠隔操作するための
仕組み。セットアップ作業はこのSSH接続を通じて行う。

**前提**: RPiと操作用PCが**同じWi-Fiルーター**に接続されていること。

接続にはRPiの「IPアドレス」(ルーター内でRPiを識別する数字の番号)が必要。
この番号は通常、起動のたびに変わるため、**先に6章の手順でIPアドレスを固定
してから**以降の作業を進める。

セットアップ用PCのターミナル(Windowsの場合はPowerShell)から以下を実行する
(`<RPiのIP>`は6章で固定したIPアドレスに置き換える)。

```bash
ssh pi@<RPiのIP>
```

「Are you sure you want to continue connecting?」と表示されたら`yes`と入力して
Enter。その後パスワードを求められたら、3.1節で設定したパスワードを入力する。

#### 3.3 必要パッケージのインストール

RPiにSSH接続した状態で以下を実行する。

```bash
sudo apt update
sudo apt install -y python3-picamera2 rpicam-apps python3-flask alsa-utils git
```

#### 3.4 カメラ・マイクの動作確認

```bash
# カメラが認識されているか確認(カメラ名が表示されればOK)
rpicam-hello --list-cameras

# サムネイル撮影テスト(test.jpgができていれば成功)
rpicam-jpeg -o test.jpg -t 1000 -n

# 動画撮影テスト(test.h264ができていれば成功)
rpicam-vid -t 5000 -o test.h264 --width 1920 --height 1080 --framerate 30 --bitrate 8000000

# USBマイクを使う場合: デバイス名を確認
arecord -l
```

`arecord -l`で表示されたデバイス名(例: `card 1: ... device 0:` →
`plughw:1,0`)は、[設定]ページの`audio_device`に設定する(8章)。


### 4. Webアプリの取得・配置

RPiにSSH接続した状態で以下を実行し、リポジトリをホームディレクトリに
取得する。

```bash
git clone https://github.com/ktak0119/RPi_video.git ~/RPi_video
```

Webアプリ本体はリポジトリルート(`~/RPi_video/`)に配置される。

#### 更新方法

機能追加・修正があった場合は、以下で最新化できる。

```bash
cd ~/RPi_video
git pull
```

5章でsystemd登録済みの場合は、更新後に以下でWebアプリを再起動する。

```bash
sudo systemctl restart rpi-webapp.service
```

`git pull`時に特定のファイルがコンフリクトする場合があるので、8章末尾・
11章のトラブルシューティングを参照。


### 5. Webアプリの起動・自動化

#### 5.1 動作確認(手動起動)

```bash
cd ~/RPi_video
./start_webapp.sh
```

起動後、6章の方法でブラウザから`http://<RPiの固定IP>:8080/`にアクセスし、
ホーム画面が表示されることを確認する。

停止する場合:

```bash
./stop_webapp.sh
```

#### 5.2 systemdへの登録(自動起動)

RPiの起動時にWebアプリが自動的に起動するように、systemdへ登録する。
9.6節の再起動・シャットダウン機能を使う場合は必須。

1. サービスファイルを**未編集のまま**コピーする
   (`~/RPi_video`内のファイルを直接編集すると、以後の`git pull`で
   コンフリクトする原因になるため、必ず先にコピーしてからコピー先を
   編集する)

   ```bash
   sudo cp ~/RPi_video/rpi-webapp.service /etc/systemd/system/
   ```

2. **3.1節で`pi`ユーザーを使い、4章で`~/RPi_video`に配置した場合は
   この手順は不要**。次の手順3へ進む。

   **別のユーザー名や配置先パスを使った場合のみ**、以下を実行してファイルを
   編集する。`User=pi`を実際のユーザー名に、`/home/pi/RPi_video`を実際の
   パスに書き換える(ホームディレクトリは`%h`ではなく絶対パスで書くこと)。

   ```bash
   sudo nano /etc/systemd/system/rpi-webapp.service
   ```

   - nanoでの保存: `Ctrl+O` → `Enter` → `Ctrl+X`(終了)

3. 登録・有効化する

   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable --now rpi-webapp.service
   ```

4. 状態を確認する(`active (running)`と表示されればOK)

   ```bash
   sudo systemctl status rpi-webapp.service
   ```

登録後は、5.1節の`start_webapp.sh`/`stop_webapp.sh`は使わず、以下を使う。

```bash
sudo systemctl restart rpi-webapp.service   # 再起動
sudo systemctl stop rpi-webapp.service      # 停止
sudo systemctl status rpi-webapp.service    # 状態確認
```

#### 5.3 システム操作に必要な権限設定

**3.1節で`pi`ユーザーを使った場合は追加設定不要**。次の章へ進む。

**`pi`以外のユーザーで運用する場合のみ**、付録Aの追加設定が必要。


### 6. RPiへのアクセス(固定IP)

「IPアドレス」とは、Wi-Fiルーター内でそれぞれの機器を識別するための
数字の番号(例: `192.168.1.100`)。通常は起動のたびに変わるが、ルーターの
設定で特定の機器に毎回同じ番号を割り当てることができる。これを「固定IP」
または「IPアドレスの固定」と呼ぶ。

RPiに固定IPを設定しておくと、毎回同じURL(`http://<固定IP>:8080/`)から
アクセスできる。設定しないと、RPiを再起動するたびにアドレスが変わり、
ブラウザからアクセスできなくなる場合があるため、**必ず設定する**。

複数台のRPiを運用する場合は、台数分それぞれに異なる固定IPを割り当てる。
こうすることで、タブレットから台ごとに別々のURLでアクセスして使い分けられる。

#### 6.1 IPアドレスの番号の決め方

IPアドレスは例えば`192.168.1.201`のような4つの数字の組み合わせで表される。
ルーターの設定で自由に割り当てられるのは末尾の数字(1〜254の範囲)。

複数台を管理する場合、末尾の数字をRPiの番号に対応させておくと覚えやすい。

| RPi | 固定IPの例 | アクセスURL |
|---|---|---|
| 1号機 | `192.168.1.201` | `http://192.168.1.201:8080/` |
| 2号機 | `192.168.1.202` | `http://192.168.1.202:8080/` |
| 3号機 | `192.168.1.203` | `http://192.168.1.203:8080/` |

> `192.168.1.`の部分はルーターによって異なる(例: `192.168.0.`や`192.168.8.`など)。
> 実際の値はルーターの管理画面で確認する。末尾の数字だけ自分で決めればよい。

### 8. 撮影設定([設定]ページ)

ナビゲーションの「設定」から、撮影内容を変更できる。設定は**次回の撮影
開始時から**反映される(撮影中の動画やプレビューには影響しない)。

| 項目(画面表示) | 内部のキー名 | 内容 | 設定可能範囲 |
|---|---|---|---|
| 横解像度 | width | 動画・サムネイルの幅(px) | 1-2400 |
| 縦解像度 | height | 動画・サムネイルの高さ(px) | 1-2400 |
| フレームレート | framerate | 動画の1秒あたりのフレーム数(fps) | 1-40 |
| ビットレート | bitrate | 動画のビットレート(bps)。画質・ファイルサイズに影響 | 100000-25000000 |
| 1ファイルの撮影時間 | video_duration | 1本の動画の長さ(秒) | 1-3600 |
| 撮影ファイル数の上限 | video_number | 何本撮影したら終了するか | 1-1000 |
| 音声入力デバイス | audio_device | USBマイクのALSAデバイス名(例: `plughw:1,0`。3.4節参照) | 文字列 |

#### bitrateの目安

| 解像度 | フレームレート | bitrate目安 |
|---|---|---|
| 1920x1080 | 30fps | 6,000,000〜10,000,000 (6〜10Mbps) |
| 1280x720 | 30fps | 3,000,000〜5,000,000 (3〜5Mbps) |
| 640x480 | 30fps | 1,000,000〜2,000,000 (1〜2Mbps) |

画質を上げたい場合はbitrateを大きく、ファイルサイズを抑えたい場合は
小さくする。

#### 補足: 設定ファイルについて

設定内容は`~/RPi_video/imageSetting.txt`に保存される
(変更前の内容は同じ場所に`.bak`として残る)。このファイルはリポジトリに
含まれているため、[設定]ページで保存した後に`git pull`を実行すると、
このファイルがローカルで変更済みであることが原因で取得できない旨の
メッセージが表示されることがある。その場合は次の手順で対処する。

```bash
cd ~/RPi_video
git checkout -- imageSetting.txt
git pull
```

(機体側で保存した設定値は`git checkout`で失われるため、必要であれば
事前に控えておき、`git pull`後に[設定]ページで設定し直す)


### 9. 操作方法(詳細)

ナビゲーションは画面上部に常に表示される: 「ホーム」「プレビュー」
「データ」「設定」「システム」。

#### 9.1 ホーム

現在の状態(待機中/プレビュー中/撮影中/停止処理中)を表示する。撮影中は
3秒ごとに自動更新される。

- 「プレビューを見る」: プレビュー画面へ(9.2節)
- 「撮影を開始する」(待機中のみ表示): 撮影開始画面へ(9.3節)
- 「撮影を停止する」(撮影中のみ表示): 確認後、撮影を停止する(9.4節)

画面下部の「RPi側 VERSION」は、`~/RPi_video/VERSION`の内容
(導入したWebアプリのバージョン)を表示する。

#### 9.2 プレビュー

カメラの映像をリアルタイムに表示する(640x480・5fps固定。撮影時の画質
設定とは独立)。撮影中は利用できない。

プレビュー画面には「ライブ」と「静止画」の2つのモードがある。

- **ライブ**: 映像をリアルタイムに流し続ける方式。スマートフォンや
  タブレットのブラウザでは、ブラウザの仕様により数十秒後に映像が
  止まることがある(ブラウザとの相性問題)
- **静止画(1秒更新)**: 1秒ごとに最新の静止画を取得して表示する方式。
  ライブに比べて動きはカクカクするが、どの端末・ブラウザでも安定して
  動作する

スマートフォン・タブレットでライブ映像がフリーズした場合は、画面上の
「静止画モードに切替」ボタンを押して静止画モードへ切り替える。

「カメラの初期化に失敗しました」と表示される場合は、9.6節の
「カメラプロセスをリセット」を試す。

#### 9.3 撮影を開始する

「撮影を開始する」から以下を入力する。

- **撮影対象名**(必須): フォルダ名`{撮影日}_{対象名}`の一部になる。
  同じ名前のフォルダが既に存在する場合はエラーになるので、別の名前を
  入力する
- **メタデータ**(任意): 撮影場所・撮影者・対象種・天気などのメモ。
  `memo.txt`としてフォルダ内に保存される(RPiのホスト名も自動で記録)
- **音声も録音する**: USBマイクが接続されている場合のみチェック

入力後「撮影を開始する」を押すと撮影が始まり、ホーム画面に戻る。

#### 9.4 撮影を停止する

ホーム画面で「撮影を停止する」を押すと、確認後ただちに停止処理が始まる。
状態は「停止処理中...」と表示され、現在実行中の動画撮影プロセスが終了する
タイミングで「待機中」に戻る。「停止処理中...」のまま画面が変わらない
場合でも、もう一度「撮影を停止する」を押すか、「キャンセルして状態に
戻る」リンクでホーム画面に戻ってよい(裏側の停止処理は継続している)。

#### 9.5 データ

撮影済みフォルダの一覧をサムネイル付きで表示する。フォルダを選択すると、
メモの内容・サムネイル一覧・ファイル一覧(サイズ付き)が表示される。

- 各ファイルはタップ/クリックでダウンロードできる
- 「このフォルダを削除する」: フォルダ全体を削除する(確認あり、元に
  戻せない)。撮影中のフォルダは削除できない

#### 9.6 システム

| 機能 | 内容 | 撮影中の制限 |
|---|---|---|
| カメラプロセスのリセット | `rpicam-vid`/`rpicam-jpeg`/`arecord`を強制終了する。プレビューが「カメラの初期化に失敗しました」になる場合の復旧手段 | 使用不可 |
| 再起動 | 1分後にRPiを再起動する。5.2節でsystemd登録済みならWebアプリも自動的に復帰する | 使用不可 |
| シャットダウン | 1分後にRPiをシャットダウンする(完全に電源OFF)。**再度使うには物理的に電源を入れ直す必要がある**(リモートでの起動は不可) | 使用不可 |
| 予約をキャンセル | 再起動・シャットダウンの予約を取り消す | 制限なし |
| 時刻設定(端末の時刻) | 操作端末(ブラウザ)の現在時刻でRPiの時計を設定する | 使用不可 |
| 時刻設定(手動入力) | 年/月/日/時/分/秒を個別入力してRPiの時計を設定する。端末の時刻が正しくない場合のフォールバック | 使用不可 |

再起動・シャットダウンには確認ダイアログがあり、実行は1分後のため、誤って
押した場合は「予約をキャンセル」で取り消せる。シャットダウンは観察終了後に
バッテリー消費を抑えて機体を回収する場合などに使う。

時刻がずれているとファイル名のタイムスタンプもずれるため、撮影開始前に
画面に表示されるRPi本体の時刻を確認し、必要であれば時刻設定を行う。


### 10. 撮影データの取り扱い

#### 10.1 ファイル命名規則

```
{フォルダ名}_{連番4桁}_{タイムスタンプ}.拡張子
```

例: `20260612_クマ_0001_20260612120000.jpg` / `.h264` / `.wav`

- フォルダ名: `{撮影開始日}_{撮影対象名}` (9.3節で入力した名前)
- 連番: `0001`から開始
- タイムスタンプ: `YYYYMMDDHHMMSS`(撮影開始時刻)
- 音声を録音しない設定の場合、`.wav`は作成されない(`.jpg`/`.h264`のみ)

#### 10.2 PC側でのMP4統合について

撮影後、フォルダ内の`.h264`(動画)と`.wav`(音声)は同じ連番・タイムスタンプ
を持つファイル名で対応している。

例: `20260612_クマ_0001_20260612120000.h264` と
    `20260612_クマ_0001_20260612120000.wav` → 同じ撮影区間の動画と音声

PC側でこれらのペアを探し、動画編集ソフト等でMP4に統合する。


### 11. トラブルシューティング

| 症状 | 確認・対処 |
|---|---|
| ブラウザでWebアプリにアクセスできない | RPiの電源・Wi-Fi接続を確認。`sudo systemctl status rpi-webapp.service`(5.2節)でWebアプリが起動しているか確認 |
| プレビューのライブ映像がフリーズする | スマートフォン・タブレットのブラウザではブラウザの仕様により映像が止まることがある。「静止画モードに切替」ボタンを押して静止画モードへ切り替える(9.2節) |
| プレビューで「カメラの初期化に失敗しました」と表示される | 9.6節「カメラプロセスをリセット」を実行する |
| 撮影開始直後に失敗する | プレビューを開いたままになっていないか確認(「カメラプロセスをリセット」で復旧) |
| 音声が記録されない | `arecord -l`(3.4節)でUSBマイクが認識されているか、[設定]の`audio_device`(8章)が正しいか確認 |
| RPiの時刻がずれている | 9.6節「時刻設定」で端末の時刻に合わせる、または手動入力する |
| [システム]の再起動・シャットダウン・時刻設定で「実行しました」と表示されるが何も起きない | 5.3節・付録Aのsudoers設定を確認。`sudo journalctl -u rpi-webapp.service -n 10 --no-pager`で`sudo: a password is required`が出ていないか確認 |
| シャットダウン後、RPiにアクセスできなくなった | 想定通りの動作。物理的に電源を入れ直す |
| 再起動後、ブラウザでアクセスできない | `sudo systemctl status rpi-webapp.service`でsystemdに登録・有効化されているか確認(5.2節) |
| `git pull`で`rpi-webapp.service`や`imageSetting.txt`がコンフリクトする | 5.2節・8章末尾の補足を参照。`git checkout -- <ファイル>`で元に戻してから`git pull`する |


### 付録A: pi以外のユーザーで運用する場合の追加設定

`pi`ユーザーであればこの設定は不要。

`pi`以外のユーザーで運用する場合、9.6節の再起動・シャットダウン・時刻設定が
正しく動作しない。これらの機能は内部で管理者コマンド(`sudo`)を実行するが、
`pi`以外のユーザーはデフォルトでその権限を持たないため。設定していない場合、
一時的に動作しているように見えても後で突然動かなくなることがあるため、
`pi`以外のユーザーで運用する場合は必ず以下を設定する。

```bash
sudo visudo -f /etc/sudoers.d/rpi-webapp
```

エディタが開いたら以下の1行を追加して保存する(`<user>`は実際の
ユーザー名)。

```
<user> ALL=(ALL) NOPASSWD: /usr/sbin/shutdown, /usr/bin/date
```

- nanoの場合: `Ctrl+O` → `Enter`(保存) → `Ctrl+X`(終了)
- viの場合: `Esc` → `:wq` → `Enter`
- `visudo`は保存時に文法チェックを行うので、エラーが出た場合は指示に
  従って修正する

設定後、`sudo -n shutdown -h +1 && echo OK`でパスワード無しで実行できるか
確認できる(`sudo -n`はパスワード入力が必要な場合エラーになる)。`OK`が
表示されたらすぐに`sudo -n shutdown -c`でキャンセルする。
